◆ update_offs

std::unordered_map<std::string, size_t> openage::renderer::opengl::GlUniformInput::update_offs

We store uniform updates lazily. They are only actually uploaded to GPU when a draw call is made. Update_offs maps the uniform names to where their value is in update_data in terms of a byte-wise offset. This is only a partial valuation, so not all uniforms have to be present here.
